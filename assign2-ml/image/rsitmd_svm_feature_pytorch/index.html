SVM + ResNet50 Features

A two-stage classifier: first extract image embeddings using a pretrained ResNet50 backbone, then train an SVM on the fixed feature vectors. This page makes the extractor → classifier split explicit.

Pipeline
Input
RSITMD image crops
The image branch is processed as a classification dataset, not a captioning task.
Preprocess
Resize + normalize + cache
Images are transformed to 224×224 tensors and cached for feature extraction.
Backbone
ResNet50 extractor
The FC head is replaced with Identity to produce dense features.
Head
RBF SVM
A classical classifier consumes the fixed embeddings from the extractor.
Output
Class predictions + report
Accuracy, classification report, confusion matrix, and saved artifacts.
What makes it different
Hybrid classical pipeline
This model separates deep feature extraction from classical ML classification, making it useful as a strong non-end-to-end baseline.
